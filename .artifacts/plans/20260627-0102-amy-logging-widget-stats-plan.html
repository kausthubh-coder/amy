Goal

Fix the user-reported Amy regressions shown in the attached screenshots: photo+prompt should add a visible log line, widgets should use the same calorie fire language and resize sanely, macro progress should show real progress instead of green-only states, streaks should represent contiguous streaks instead of total logged days, Stats/Streaks should have simpler top navigation, streak details should show actual eaten totals, and Today note calories should stay aligned while scrolling/editing.

Current Findings

  • CaptureModal logs image estimates through addEntryFromDraft; the store appends a note line but skips duplicates, so a caption matching an existing typed line can count calories without changing visible text.
  • StatsModal uses activeDays.size, which counts all logged days, not a break-sensitive streak.
  • The Stats modal content starts with another title-like Stats tab below the modal shell header, creating the duplicate title/X/tabs layout in the screenshot.
  • MacroRing only changes the border when progress exceeds 80%, so rings look broken and mostly inactive.
  • Today's note input and calorie rail are separate absolute layers. The input scrolls internally on longer notes, while the rail does not, causing misalignment.
  • The widget XML uses fixed icon/action widths and emoji glyphs. It sets resize metadata but the layouts do not adapt well to smaller widget sizes.

Implementation Split

  • Main thread: create artifacts, patch shared store behavior, Today alignment, stats/streaks UI, macro rings, widget plugin, and release/build prep.
  • Background audit A: read-only review of photo/gallery logging and prompt-to-line behavior.
  • Background audit B: read-only review of Android widget emoji/resize/deep-link behavior.
  • Background audit C: read-only review of Stats/Streaks, macro progress, streak semantics, and Today row alignment.

The environment exposes private multi-agent tools, not the user-visible Codex app thread API described by Oracle. Parallel review is still being done with separate agents and integrated in this orchestrator thread.

Planned Changes

  • Add an explicit store helper for adding image/barcode/saved entries while ensuring the note line appears even if it duplicates an existing line.
  • Make the Today note input non-scrollable and align the calorie rail with measured wrapped line heights.
  • Compute current contiguous streak from today/selected day backward, show total logged days separately, and show consumed/goal summaries in Streaks.
  • Move Stats/Streaks tabs into the modal title area by making the modal shell title neutral and letting the segment lead the sheet content.
  • Replace static macro ring borders with SVG progress rings and distinct macro colors.
  • Update widget layouts to use the flame calorie marker consistently, use consumed labels with units, reduce fixed widths, add min/max resize hints, and keep action cells compact.

Verification Strategy

  • Run npm test for TypeScript.
  • Run npm run check:deps and npx expo-doctor for release sanity.
  • Run Android prebuild in a temp/generated native path and inspect widget XML/manifest outputs.
  • Install and smoke-test on an Android emulator if available; use Computer Use for visible app checks where the emulator UI is reachable.
  • Prepare a local APK and attempt an EAS preview build submission using the configured Expo project if credentials are available.
